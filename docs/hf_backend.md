# HF Backend

## Overview

The HF backend runs real generation through `transformers` and can execute in three cache modes:

- `baseline`
- `mse`
- `turboquant`

`mse` and `turboquant` perform compressed-cache analysis on real `past_key_values` generated by the model during prefill/decode.

## Commands

Generation:

```bash
tqjax-generate \
  --backend hf \
  --model Qwen/Qwen3.5-2B \
  --cache turboquant \
  --bits-k 3 \
  --bits-v 2 \
  --prompt "Explain this in plain English." \
  --max-new-tokens 64
```

Benchmark:

```bash
tqjax-bench \
  --backend hf \
  --model Qwen/Qwen3.5-2B \
  --contexts 2048,4096,8192,16384,32768 \
  --cache-modes baseline,mse,turboquant \
  --max-new-tokens 32
```

## Metrics

Each run exports:

- `ttft_s`
- `prefill_tps`
- `decode_tps`
- `wall_time_s`
- `prompt_tokens`
- `generated_tokens`
- `kv_cache_bytes` (when compressed analysis mode is used)
- `compression_ratio` (when compressed analysis mode is used)
- `quality_score` (score-vector cosine estimate in compressed analysis mode)

## Practical Notes

- Use `--device cpu` for deterministic fallback on CPU-only machines.
- Use `--max-eval-layers` to reduce compressed-cache analysis overhead during smoke tests.
- In this milestone, HF cache compression is integrated as runtime analysis on real generation traces. Additional direct model-cache replacement paths remain future work.
